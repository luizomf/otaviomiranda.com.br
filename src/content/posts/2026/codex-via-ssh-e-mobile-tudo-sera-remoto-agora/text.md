Será que agora estamos seguindo para um modelo mais remoto? Eu já tinha migrado
a maioria dos meus projetos para uma espécie de trabalho remoto. Com containers
do Docker.

Porém, agora parece que todas as empresas, como Anthropic, OpenAI, Google e
outras, também estão para o trabalho mais remoto (digo, em servidores remotos).

Neste vídeo, falo sobre o Codex App, que agora tem a versão mobile. E essa
versão mobile não significa que o Codex vai rodar no seu telefone. Isso seria
inútil pra qualquer dev sério.

O Codex vai poder controlar um dispositivo remoto através da versão mobile. O
servidor continua sendo o `app-server` do Codex CLI via SSH ou a versão
conectada do Codex App.

Isso significa que, a partir de agora, você vai poder deixar um projeto rodando
em qualquer lugar e, se o seu smartphone estiver conectado neste projeto,
independente de onde você estiver, você vai conseguir controlar este projeto.

Por exemplo, suponha que você saiu para ir à farmácia enquanto alguma coisa
estava acontecendo no seu projeto. Um agente iniciou uma task de longa duração e
você não pôde esperar terminar.

Se este agente terminar o serviço, ele vai te notificar. Isso já acontece no
próprio CLI ou no Codex App, mas na hora que ele terminar, você também vai
receber uma notificação no seu telefone e poderá dar sequência no trabalho
utilizando o Codex Mobile. A parte mobile de verdade continua lá com o "ChatGPT"
na versão mobile. A parte séria da coisa é o Codex que estou falando.

Enfim, melhor você assistir ao vídeo. É curtinho e já demonstra o que você pode
fazer com o Codex Mobile e Codex com SSH Remoto.
