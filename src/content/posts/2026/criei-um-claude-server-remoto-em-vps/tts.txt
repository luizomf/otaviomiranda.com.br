Já faz algum tempo que o Claude Remote Control existe, mas eu sempre esbarrava no mesmo problema: contexto.

Você pode usar o comando Claude remote control para iniciar uma thread controlada por outros apps do Claude.
Isso inclui Claude Desktop, Claude Web e Claude Mobile.
O [S IY1 EH1 L AY1] passa a funcionar como um servidor.
E suas mensagens ficam sincronizadas entre todos os dispositivos conectados.

Isso perde um pouco o propósito se você forçar.

Por exemplo: se eu inicio uma sessão com remote control e saio para fazer alguma coisa, só tenho aquela única thread disponível para tudo.
E outra: só no computador onde iniciei o Claude Code, que é o [S IY1 EH1 L AY1].

Creio que a ideia inicial disso era só para você terminar algum serviço sem ter que ficar preso no computador.

Esse comportamento força você a deixar o computador ligado.
Também precisa manter o Claude Code aberto e usar essa única thread, gastando cada vez mais tokens a cada mensagem.

Fora o perigo do Context Rot, que também entra nessa conta.
Sabemos que, quanto mais você mantém um único chat, maior a chance de poluir o contexto com alguma coisa.

Se acontecer, é muito difícil o agente se recuperar sem que você limpe o contexto.
Spoiler: você não consegue limpar o contexto nem iniciar uma nova thread de dentro de outra thread via remote control.

Depois de muito pensar, bater cabeça e tentar criar uma nova thread do Claude remotamente, lembrei do meu querido tê mux.
Nem tentei tanto assim, isso é só para dar ênfase no texto.

Para quem nunca usou o tê mux, ele é um multiplexador de terminal.

Basicamente, ele te dá a habilidade de organizar sessões, janelas e painéis com vários shells.

Quando você vê alguém com terminais divididos na tela, é muito provável que essa pessoa esteja usando um multiplexador como o tê mux.
Ou um desses terminais modernos, tipo Ghostty.

Eu sei que você achou que eu não estava digitando mesmo, mas vou te perdoar dessa vez.
A imagem mostrava o tê mux usando painéis para dividir uma janela dentro de uma sessão.
Também aparecia um trecho do meu texto, que eu acabei de corrigir e ficou diferente.
E ainda tinha uma prova de que a I A não tem a mínima noção sobre o que é uma piada.
Era uma piada do Qwen. Eu nem entendi o propósito ali.

Enfim, não estou te dando um curso de tê mux nem falando para você usar isso.
Mas use, vai por mim, fica a dica não solicitada.

O ponto é que as sessões precisam de uma janela, e essa janela precisa de um shell.
Isso é o que permite que o tê mux rode vários terminais em background.
Fora buffers e outras coisas que nem vou falar, porque eu nem sei.

O que eu pedi para o Claude fazer foi testar com tê mux.

Quando quero que ele inicie uma nova thread no Claude Code, ele cria uma sessão no tê mux.
Essa sessão entra com uma janela, e dentro da janela fica um shell rodando o Claude em modo remote control.
